# Global Supermarket Sales Analysis (Power BI)

## Overview
This project analyzes transactional sales data from a multinational supermarket to identify profitability drivers, return risk factors, and customer satisfaction issues.  

The goal is to provide data-driven recommendations to reduce returns, improve operational efficiency, and increase profitability.

## Dataset
- 5,500 transaction records  
- Sales, revenue, and product data  
- Customer demographics  
- Shipping and delivery details  
- Return status and return reasons  
- Customer ratings  

## Tool Used
- Power BI (Power Query, Data Modeling, DAX, Dashboard Design)

## Project Steps
1. Loaded dataset into Power BI  
2. Cleaned and transformed data (handled missing values, standardized fields)  
3. Performed Exploratory Data Analysis (EDA)  
4. Created KPIs (Revenue, Return Rate, Avg Rating, Avg Purchase)  
5. Built interactive dashboard  
6. Generated business insights and recommendations  

## Dashboard Highlights
- Revenue Overview  
- Return Rate Analysis  
- Customer Rating Trends  
- Product Category Performance  
- Shipping Mode Comparison  
- Region-wise Performance  

## Key Results
- High return rate (~50%) impacting profitability  
- Low average customer rating (~3.0)  
- Quality issues and late delivery are major return drivers  
- Operational inefficiencies contribute to revenue leakage  

## How to Run
1. Open the `.pbix` file in Power BI Desktop  
2. Refresh data if needed  
3. Explore dashboard using filters and slicers
